📥 Model Download & Storage Flow

Where Models Are Downloaded:

Source: HuggingFace Model Repository

Storage Location: FileSystem.documentDirectory + 'models/'

File Path Example: /data/user/0/com.yourcompany.offlinegpt/files/models/qwen2.5-0.5b-instruct-q4_0.gguf

Download Process (Step-by-Step):

Step 1: User selects model from ModelSelectScreen
⬇️
Step 2: ModelContext.downloadModel() triggered
⬇️
Step 3: FileSystem.createDownloadResumable() starts download from HuggingFace
⬇️
Step 4: Progress callback updates UI (0-100%)
⬇️
Step 5: File saved to device storage in GGUF format
⬇️
Step 6: Model metadata stored in AsyncStorage
// Example Download Code (ModelContext.js) const downloadModel = async (model) => { const modelDir = FileSystem.documentDirectory + 'models/'; const modelPath = modelDir + model.fileName; // Create download resumable const downloadResumable = FileSystem.createDownloadResumable( model.downloadUrl, // HuggingFace URL modelPath, {}, (progress) => { const percent = (progress.totalBytesWritten / progress.totalBytesExpectedToWrite) * 100; setDownloadProgress(percent); } ); await downloadResumable.downloadAsync(); await AsyncStorage.setItem('downloaded_model', model.id); };

🧠 How AI Inference Works (Most Important)

⚡ The Core AI Engine: llama.rn

What it does: Runs LLM inference directly on the mobile device CPU/GPU

Based on: llama.cpp (C++ library for efficient LLM inference)

Model Format: GGUF (quantized models optimized for mobile)

Why it works offline: All computation happens on-device, no API calls needed

Inference Flow (When User Sends Message):

Step 1: User types message in ChatScreen
⬇️
Step 2: ChatContext.sendMessage() called
⬇️
Step 3: Check if LLM model is loaded in memory
⬇️
Step 4: If not loaded, initializeLlama() loads model from storage
⬇️
Step 5: llamaContext.completion() generates response
⬇️
Step 6: AI response streamed back to UI in real-time
// AI Initialization (ChatContext.js) const initializeLlama = async () => { const { initLlama } = await import('llama.rn'); const modelPath = FileSystem.documentDirectory + 'models/' + selectedModel.fileName; const context = await initLlama({ model: modelPath, // Path to downloaded GGUF file n_ctx: 2048, // Context window size n_batch: 512, // Batch size for processing n_threads: 4, // CPU threads to use use_mlock: true, // Lock memory for performance use_mmap: true, // Memory-map file for efficiency }); return context; }; // Generate AI Response const sendMessage = async (userMessage) => { const response = await llamaContext.completion({ messages: [ { role: 'system', content: 'You are a helpful assistant.' }, { role: 'user', content: userMessage } ], n_predict: 512, // Max tokens to generate temperature: 0.7, // Creativity (0-1) top_p: 0.9, // Nucleus sampling }); return response.text; };

🔄 Context Providers (State Management)

Purpose: User authentication (Google)

Storage: AsyncStorage + llama.rn

🔨 Build Process

Development vs Production:

Aspect Development (Expo Go) Production (Standalone APK)
Testing Method Expo Go app on phone Install APK directly
AI Functionality Mock responses only Real AI with llama.rn
Build Command npx expo start eas build --platform android
Hot Reload ✅ Yes ❌ No (rebuild needed)
Native Modules ❌ Not supported ✅ Fully supported

Why llama.rn Only Works in APK:

llama.rn is a native module that requires C++ compilation. Expo Go is a generic sandbox app that doesn't include custom native code. Therefore:

  • Development testing uses mock AI responses
  • Production APK includes compiled llama.rn native library
  • EAS Build compiles C++ code and creates standalone APK

📊 Complete Data Flow Diagram

USER INTERACTION ↓ [ChatScreen UI] ↓ ChatContext.sendMessage() ↓ Check if model loaded? ──NO──→ initializeLlama() ↓ ↓ YES Load from FileSystem ↓ ↓ ↓ ←──────────────────────────┘ ↓ llamaContext.completion() ↓ [llama.rn Native Module] ↓ [llama.cpp C++ Engine] ↓ [GGUF Model File Processing] ↓ [Token Generation on Device CPU/GPU] ↓ Response Text Streamed Back ↓ ChatContext Updates State ↓ UI Re-renders with AI Response ↓ Save to AsyncStorage (chat history)

⚡ Performance Optimization

Model Quantization

Models use Q4_0 quantization (4-bit) reducing size by 75% with minimal quality loss

Memory Mapping

use_mmap: true loads model without consuming RAM entirely

Multi-threading

n_threads: 4 utilizes multiple CPU cores for faster inference

Context Caching

Model stays loaded in memory between messages for instant responses
