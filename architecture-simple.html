🤖 OfflineGPT - Quick Architecture Overview

⚡ What is OfflineGPT?
A mobile app that runs AI (ChatGPT-like) completely offline on your phone. No internet needed after model download. All processing happens on your device.

🛠️ Main Technologies Used

Technology Purpose Why Important
React Native Mobile App Framework Cross-platform (Android + iOS)
Expo SDK 51 Build System Easy development & APK building
llama.rn AI Engine 🔥 Runs LLM on device (MOST CRITICAL)
AsyncStorage Data Storage Saves chats, settings locally
FileSystem File Management Downloads & stores AI models

🧠 How AI Works in the App

Most Important Package: llama.rn
This is a React Native wrapper for llama.cpp (a C++ library). It allows running Large Language Models directly on mobile devices without any cloud/internet connection.
Complete Flow When User Sends a Message:
1️⃣ User types message in Chat Screen
⬇️
2️⃣ App checks if AI model is loaded in memory
⬇️
3️⃣ If not loaded: Load model from phone storage using llama.rn
⬇️
4️⃣ llama.rn processes message using phone's CPU/GPU
⬇️
5️⃣ AI generates response (takes 3-10 seconds)
⬇️
6️⃣ Response displayed in chat + saved to storage

📥 Where & How Models are Downloaded

Download Source: HuggingFace (public AI model repository)
Storage Location: Phone's internal storage at /data/.../files/models/
File Format: GGUF (optimized for mobile devices)
Size Range: 100MB - 2.3GB depending on model
Model Download Process:
1️⃣ User opens "Model Selection" screen
⬇️
2️⃣ Sees 7 available models with sizes & descriptions
⬇️
3️⃣ Taps "Download" on a model (e.g., Qwen 2.5 0.5B - 400MB)
⬇️
4️⃣ FileSystem downloads from HuggingFace with progress bar
⬇️
5️⃣ Model saved to phone storage as .gguf file
⬇️
6️⃣ Model info saved in AsyncStorage (metadata)
⬇️
7️⃣ Ready to use in Chat! Works 100% offline

🧪 Development vs Production

⚠️ Important Note:
Real AI (llama.rn) only works in standalone APK, not in Expo Go app during development. This is because llama.rn requires native C++ code compilation.
Aspect Development (Expo Go) Production (APK)
Testing npx expo start → Scan QR Install APK on phone
AI Works? ❌ No (mock responses) ✅ Yes (real llama.rn)
Hot Reload? ✅ Yes ❌ No
Build Command npx expo start eas build --platform android

❓ Quick Answers to Lead's Questions

Q: Where are models downloaded from?
A: HuggingFace repository (public AI model hub)
Q: Where are models stored on device?
A: FileSystem.documentDirectory + 'models/' (app's private storage)
Q: How does AI run offline?
A: llama.rn (React Native wrapper for llama.cpp C++ library) processes everything on device CPU/GPU
Q: What's the main AI package?
A: llama.rn version 0.3.2 - This is the CORE technology
Q: How big are the models?
A: 100MB to 2.3GB (recommended: 400MB Qwen 2.5 for 4GB RAM phones)
Q: How fast is inference?
A: First message: 10-15 seconds (loading model). After that: 3-5 seconds per response
Q: Can it run on 4GB RAM phone?
A: Yes! Use smaller models (0.5B - 1.5B parameters, 400MB - 1.3GB size)
Q: Why use React Native?
A: Cross-platform (one codebase for Android + iOS), large community, llama.rn support
Q: How is APK built?
A: EAS Build (Expo cloud build service) compiles native code + creates APK
Q: Does it send data to cloud?
A: No! 100% offline. All data stays on device. No tracking, no analytics.
🎯 Summary in 3 Points:

1. Technology Stack: React Native + Expo + llama.rn (AI engine)

2. How AI Works: Models downloaded from HuggingFace → Stored on phone → llama.rn processes messages using phone's CPU/GPU → 100% offline

3. Main Achievement: Successfully created a ChatGPT-like app that works completely offline on mobile devices with 4GB RAM, no internet required after initial model download.
